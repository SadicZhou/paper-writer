## LLM Provider: reasoning/thinking delta collection
Fixed `packages/core/src/llm/provider.ts` to collect `thinking_delta` and `reasoning_text.delta` events across all 3 streaming paths (Anthropic SSE, OpenAI Responses SSE, pi-ai unified). Without this, reasoning models (DeepSeek-R1, Claude extended thinking) would throw "LLM returned empty response from stream" because usage tokens were consumed but no `text_delta` events were produced.

## AIGC Output Quality Hardening
Created `packages/core/src/utils/ai-text-cleaner.ts` — `stripAIMetaText()` strips AI conversational prefixes, meta-section headers, and instruction echoes from LLM outputs.

Created `packages/core/src/utils/markdown-repair.ts` — `repairMarkdownStructure()` recovers figure markers (`【图X】`), headings, and Mermaid blocks lost during AIGC reduction rewrites.

Updated `academic-polisher.ts` and `ai-reduction-reviser.ts`:
- Hardened `parsePolishOutput()` and `parseRevisionResponse()` fallback parsers to skip AI conversational lines
- Replaced vague "不要格式化" prompt instruction with explicit Markdown protection rules (headings, figure markers, Mermaid blocks, bold/italic, citations, tables)
- Wired `repairMarkdownStructure()` as post-processing after each revision iteration

## Outline Validation Repair
Created `packages/core/src/utils/outline-repair.ts` — `repairSectionNumbers()` auto-assigns section numbers for sections with empty `number` fields before Zod validation. Applied in both `outline-builder.ts` and `outline-agent.ts`.

**Why:** LLM occasionally returns sections with empty `number` fields, failing `z.string().min(1)` validation at indices 8-10 of the sections array.
